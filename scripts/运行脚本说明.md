# 运行脚本说明（`scripts/`）

本文件说明 `scripts/` 下各脚本的**用途、差异**与**输出产物位置**。

## 通用输出目录（最重要）

- **训练/评测/量化产物**：`artifacts/{model}/{train_task}/{optimizer}/{seed}/{adaptation}/...`
  - Full-FT checkpoint：`.../full_ft/fp/`
  - LoRA/QLoRA checkpoint：`.../lora/` 下对应目录
  - PTQ 产物：`.../full_ft/ptq/{track}/`（如 `w8a8_smoothquant`、`w4a16_omniquant`、`w4a4_omniquant`）
  - lm-eval 原始输出：一般在对应模型目录下的 `lm_eval_out/`
  - 评测分数 JSON：
    - FP：`.../metrics_fp.json`
    - PTQ：`.../ptq/{track}/metrics_quant.json`
- **汇总 CSV**（aggregate 输出）：`results/`
  - RQ1 阶段 A（FP-only）：`results/rq1_full_ft_fp.csv`
  - RQ1 阶段 B（FP vs PTQ）：`results/rq1_full_ft_ptq.csv`

> 注：`train_task` 指训练任务名（例如 `metamath`、`codefeedback`），但 FP/量化后的评测任务由 `configs/tasks.yaml` 的 `eval_task` / `lm_eval_task` 决定（例如 `metamath -> gsm8k`）。

## RQ1（已拆分为两阶段）

### 四个脚本的区别（你问的这四个）

| 脚本 | 类型 | 做什么 | 前置条件 | 主要输出 |
|------|------|--------|----------|----------|
| `run_rq1_train_smoke.sh` | **smoke** | **阶段 A**：训练（Full-FT）→ FP `lm-eval` → 聚合 FP CSV | 需要已准备数据（`python -m exp.data.prepare_all`） | `artifacts/.../full_ft/fp/`、`metrics_fp.json`、`results/rq1_full_ft_fp.csv` |
| `run_rq1_train_matrix.sh` | **matrix** | **阶段 A 批量**：遍历 models×tasks×opts×seeds 跑 `run_rq1_train` | 同上 | 同上（批量） |
| `run_rq1_ptq_smoke.sh` | **smoke** | **阶段 B**：PTQ（SmoothQuant/OmniQuant）→ PTQ `lm-eval`（各 track）→ 聚合 PTQ CSV | **必须先完成阶段 A**（需要 FP checkpoint） | `artifacts/.../ptq/{track}/...`、`metrics_quant.json`、`results/rq1_full_ft_ptq.csv` |
| `run_rq1_ptq_matrix.sh` | **matrix** | **阶段 B 批量**：遍历 models×tasks×opts×seeds 跑 `run_rq1_ptq` | 同上 | 同上（批量） |

#### smoke vs matrix（核心差别）
- **smoke**：只跑单个配置（你用环境变量设 `MODEL/TASK/OPTIMIZER/SEED`），用于快速验证流程。
- **matrix**：批量遍历一组模型/任务/优化器/随机种子，跑完整实验矩阵。

#### 阶段 A vs 阶段 B（核心差别）
- **阶段 A**只负责：训练 + FP 评测 + FP 汇总表。
- **阶段 B**只负责：PTQ + 量化后评测 + FP vs PTQ 汇总表（依赖阶段 A 的 FP checkpoint）。

## 快速训练（不走 RQ1 的 aggregate / PTQ）

- `run_train_smoke.sh`
  - 用途：一条命令跑 `exp.finetune.run_axolotl`，默认训练后**自动评测**（auto-eval）。
  - 产物：同样写入 `artifacts/...`，并额外写 `metrics_auto_eval_<eval_task>.json`（位于该 run 的目录下）。
  - 适合：只想“训练+立刻看一个分数”，不需要 RQ1 的 CSV 汇总、不需要 PTQ。

- `run_eval_smoke.sh`
  - 用途：对**已有 checkpoint** 单独跑一次 `exp.eval.run_lm_eval`（FP / PTQ / QAT 取决于 `STAGE`/`TRACK`）。
  - 产物：写对应的 `metrics_fp.json` / `metrics_quant.json` 与 `lm_eval_out/`。

---

## DEBUG / 调试训练

- `debug_axolotl_train.sh`
  - 用途：单进程 in-process 方式跑 Axolotl（便于断点调试：dataset→model→train）。
  - 产物：与普通训练一样写到 `artifacts/...`（但它主要用于调试逻辑，不建议用来跑多卡/正式实验）。

- `run_rq3_matrix.sh`：RQ3（QAT 等）矩阵

## 环境/依赖类脚本

- `setup_hf_auth.sh`：配置 HuggingFace token（数据准备/下载用）
- `install_vllm.sh`：安装 vLLM wheel（加速 `run_eval_smoke.sh` / auto-eval）
- `setup_external.sh`：克隆外部仓库（SmoothQuant / OmniQuant / QA-LoRA / LoRA-One）
